## Quick Reference Checklists

### Daily Commit Message Checklist

âœ… **Before committing:**
- [ ] Message is 50 characters or less
- [ ] Starts with capital letter
- [ ] Uses imperative mood ("Add feature" not "Added")
- [ ] Explains WHAT and WHY, not HOW
- [ ] No period at end

âŒ **Don't do this:**
```
fixed stuff
updated code
changes made
bug fix
```

âœ… **Do this instead:**
```
Fix authentication timeout in session handler
Add user profile image upload feature
Refactor database query optimization
Update API documentation for endpoints
```

### Repository README Essentials

Each repository README MUST include:

- [ ] **Project Title** - Clear, descriptive
- [ ] **Overview** - What it does (2-3 sentences)
- [ ] **Problem** - What need does it solve?
- [ ] **Features** - Bullet point list (3-5)
- [ ] **Tech Stack** - Technologies used
- [ ] **Installation** - Step-by-step setup
- [ ] **Usage** - How to use the project
- [ ] **License** - MIT, Apache, GPL, etc.
- [ ] **Author** - Your name and contact
